JAVA - Mini Object-Relational Mapping for MySQL SGBDR

What is an ORM ?

Object-Relational Mapping (ORM) in computer science is a programming technique for converting data between incompatible type systems using object-oriented programming languages. This creates, in effect, a "virtual object database" that can be used from within the programming language.

Allows to manipulate databases with predefined methods.

How to use our ORM ?

  1. Create your own JAVA project (In our case we're using Eclipse IDE)
  2. Add the JAR files
    • "mysql-connector-java-5.1.6-bin.jar " MySQL Database Connector
    • "MoussaOUSTOUH-ORM-MySQL-(v1.0).jar " Our ORM JAR files
  1. Create a JAVA class to use our ORM (In our case "Main.java")
package pkg1;

import java.sql.Connection;
import java.sql.DriverManager;

import MO.ORM.mySQL.MoussaOUSTOUH;

public class Main {

	public static void main(String[] args) {
		try {
			Class.forName("com.mysql.jdbc.Driver");
			String db="jdbc:mysql://localhost:3306/test_orm", user="root", pass="";
			Connection con=DriverManager.getConnection(db, user, pass);

			//MoussaOUSTOUH.ORMv1(con, "package1");
			//MoussaOUSTOUH.ORMv1(con, "package1\\package2");
			MoussaOUSTOUH.ORMv1(con, "package1.package2");
			
		} catch (Exception e) { e.printStackTrace(); } 
	}
}
  • Create a database connection with MySQL database
    • db : database URL
    • user : username
    • pass : password
  • Then use our ORM static method as it show in the line 16, 17 or 18.
    This method has 2 arguments :
    1. Database connection that we had created in the previous step.
    2. The package where we gonna generate the modules and the DAOs
  • Run it to generate your database modules and DAOs.

Modules and DAO modules ?

After running it, 2 package will generated in the package you choose in the previous step.
  1. Modules package "module"

    Contains the representation of the database tables as JAVA classes.

    Every table attribute represent as a class attribute with the setter and the getter.

  2. DAOs package "dao"

    Contains the data access object which contains the methods to interact the object with the database.

    Every DAO class contains 5 methods
    • insert(arg1)
    • update(arg1, arg2)
    • delete(arg1)
    • find(arg1, arg2)
    • findAll(arg1, arg2)

How to use DAO methods ?

  • insert(arg1) 
package pkg1;

import java.sql.Connection;
import java.sql.DriverManager;

import package1.package2.dal.PersonDAO;
import package1.package2.module.Person;

	public static void main(String[] args) {
		try {
			Class.forName("com.mysql.jdbc.Driver");
			String db="jdbc:mysql://localhost:3306/test_orm", user="root", pass="";
			Connection con=DriverManager.getConnection(db, user, pass);
			
			Person person=new Person();
			person.setId(1);
			person.setFirst_name("Moussa");
			person.setLast_name("Oustouh");
			
			PersonDAO personDAO=new PersonDAO(con);
			personDAO.insert(person);
		} catch (Exception e) { e.printStackTrace(); }
	}
}

package pkg1;

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.ResultSet;
import java.util.Vector;

import package1.package2.dal.PersonDAO;
import package1.package2.dal.QuerySQL;
import package1.package2.module.Person;

public class Main {

	public static void main(String[] args) {
		try {
			Class.forName("com.mysql.jdbc.Driver");
			String db="jdbc:mysql://localhost:3306/test_orm", user="root", pass="";
			Connection con=DriverManager.getConnection(db, user, pass);
			
			QuerySQL querySQL=new QuerySQL(con);
			//this method could execute any request SQL, it work like a setter without return
			querySQL.exequteSQL("delete from person where id=36");

			//this method could execute any request SQL, it work like a getter with a ResultSet return
			ResultSet rs=querySQL.executeQuerySQL("select * from person");
		} catch (Exception e) { e.printStackTrace(); }
	}
}
